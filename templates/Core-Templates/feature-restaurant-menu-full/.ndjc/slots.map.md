# Slots Map (Module: feature-restaurant-menu)

本模块在模板定义的各页面插槽（slots）中的挂载方式。  

模板侧 `slots` 在 ndjc-sst 中定义了：  

- 标准插槽：`header` / `hero` / `primary` / `secondary` / `detail` / `sheet` / `tabBar`  
- 可选插槽：`fab` / `dialog` / `settings`  

本模块主要使用 `primary` / `detail` / `sheet` / `settings` 四类插槽，实现餐厅菜单业务。

## 1. 菜单列表页（routeId = home）

- 使用 `primary` 插槽作为菜单列表的主体区域：

  - 展示菜品卡片列表，按分类分组（热菜 / 冷菜 / 饮品 / 甜品等，可由 UI 包决定具体样式）；
  - 每条卡片包含：图片缩略图、中英文菜名、当前价格（优先展示折扣价）以及推荐标记；
  - 点击卡片跳转到 `detail` 路由（菜品详情页）。

- `header` / `tabBar` 等插槽仍由模板 + UI 包提供（如 AppBar、底部导航），本模块不覆盖。

- 管理员入口：
  - 在 `primary` 区域中通过按钮/菜单项提供“管理员登录”入口；
  - 该入口触发后打开挂载在 `sheet` 插槽中的登录面板（见下文）。


## 2. 菜品详情页（routeId = detail）

- 使用 `detail` 插槽展示单个菜品的详细信息：

  - 大尺寸图片；
  - 中英文菜名、描述；
  - 原价 / 折扣价（若存在折扣价，原价可使用删除线或弱化样式展示，由 UI 包决定）；
  - 推荐标记 / 售罄状态。

- `primary` 插槽在 `detail` 路由下不由本模块使用，避免与列表页语义混淆。

## 3. 管理登录（Admin Login）

- 使用 `sheet` 插槽呈现管理员登录界面：

  - 由菜单列表页的“管理员入口”触发；
  - 提供密码 / PIN 输入框与提交按钮；
  - 登录成功后关闭 `sheet`，并切换到管理界面（`settings` 插槽）。

> 说明：
> - 具体呈现形式（底部弹出 / 半屏抽屉）由 UI 包的 `BottomSheet` 等组件实现；
> - 模块只负责触发打开/关闭逻辑和状态管理。

## 4. 管理界面（Admin Dashboard）

- 使用 `settings` 插槽作为管理员菜单管理页面的主体区域：

  - 展示所有菜品列表（可排序）；
  - 支持新增菜品、编辑现有菜品、删除菜品；
  - 可编辑字段包括：中/英文名称、分类、图片、描述、原价/折扣价、售罄与推荐状态。

- 若 UI 包暂未实现 `settings` 插槽，也可以由模板降级为普通页面区域，但本模块将该能力声明为“优先挂载在 settings 插槽”。

## 5. 其他插槽

- 当前版本本模块不使用 `hero` / `secondary` / `fab` / `dialog` 等插槽；
- 若未来需要展示宣传 Banner / 推荐合集等内容，可在保持 SST 插槽语义的前提下扩展本文件。
